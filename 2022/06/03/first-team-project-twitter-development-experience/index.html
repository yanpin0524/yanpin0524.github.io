就是大家認識的 Twitter 的簡易版

開發流程與心得

第一階段：確認任務與分工

專案開始前我超級期待的，因為第一次與別人一起開發專案，直到要進行第一次開會時，我才突然變得超級緊張，因為我已經宅在家太久，很久沒有與家人以外的人說話了，結果開會時很多想說得話都卡在嘴裡說不出來，好在隊友們都很友善，所以會議還是蠻順利的，這次的會議主要是在進行 專案規格分析、設定驗收標準、規劃與分配任務。

在這次專案中我擔任的角色：

這次專案採用 “前後分離” 的實作方式，而我是負責後端的

  1. 工作內容大概是：建構 Model
  2. 實作 tweet、reply、like、follow 功能相關的 API
  3. 還有負責 Demo 作品給助教看 (這能算工作嗎…)

第二階段：專案實作

之後就開始進行專案實作了，可是就像剛剛說的我是第一次跟別人協作專案，所以在跟隊友的溝通上我還是會有些膽怯，怕造成別人的困擾，所以就算進度落後也一直裝成沒問題的樣子。

但後來我逐漸發現明明就遇到困難了，卻選擇不跟隊友溝通那才是造成別人的困擾，而且大家明明都很友善，沒什麼好怕的，所以我決定把遇到的問題跟隊友講，之後隊友也熱心地跟我一起把問題給解決了，這時我才意識到這就是團隊，互相溝通、互相協助。

小插曲１：我在寫完了，很多功能之後才發現，這跟測試檔要求的規格不一樣，結果直好全部重改，助教明明已經再三提醒了 “測試檔要先看”…“測試檔要先看”…“測試檔要先看”…，為什麼我就是不聽啊啊啊啊啊啊啊啊啊

小插曲２：我每次在聽前端的隊友講前端的東西時都像在聽外星文一樣，不知道前端在聽後端的東西時是不是一樣的想法啊？ 我真的必須去學習一下前端的知識了，不然溝通起來太辛苦了 T_T

第三階段：部署上線＆成果驗收

要開始將整個專案收尾了，讓完整的程式上線，後端的推到 heroku 上，前端則推到 github page 上，並把前後端串接，然後我的隊友真的很神 從上線到前後串接，他都幫忙做完了，我都沒有插手的餘地，而我的工作就只整剩下 修修一些小 bug 、demo 一下專案 之類的簡單工作。

小插曲：在我們準備提交專案的前一天，heroku上的資料庫突然崩潰了，這可是提交專案的前一天誒，害我超緊張的，之後我就在想怎麼突然就崩潰了，之前還好好的啊，我就在想會不會是誰去動了資料庫的設定，然後我仔細的回想後突然恍然大悟，啊…是我動了編碼設定倒置資料庫出錯的，兇手竟是我自己，之後把編碼調回去之後問題就解決了。

結語

在這次的專案中，我從緊張到習慣，學會了與團隊合作的重要性。雖然一開始有點膽怯，但透過與隊友的溝通，我逐漸克服了這種感覺。

專案中的小插曲也讓我笑了不少，特別是資料庫的編碼設定問題，居然是我自己搞出來的。這次經歷讓我更加注重細節，也明白到溝通與協作是成功的關鍵。

感謝我的團隊成員，一起度過了這段有趣的專案旅程。期待未來能繼續挑戰，更加成長！
